Хронология ключевых идей, решений и поворотов. Не техлог (для этого есть CHANGELOG.md), а *интеллектуальная история*: как менялось понимание задачи и подход к её решению.

## 2026-03 (ранний) — Зарождение: контейнеры и стенография

**Исходная идея:** Значение слова для конкретного человека = то, что он помещает внутрь через свои формулировки. Не словарное определение, а персональное содержимое.

**Ключевые решения:**
- Модель контейнера: понятие = контейнер + содержимое других контейнеров
- Стенографический принцип: записывать только то, что следует из слов, не интерпретировать
- Суждение (S→V→O + свойства) как единица записи
- Глагол — одновременно связка и понятие-контейнер

**Теоретический фундамент:** Wittgenstein (значение = употребление), Frege (sense/reference), Kelly (личные конструкты), Neo-Davidsonian event semantics.

**Почему именно так:** Конфликты и непонимание между людьми — вопрос определений. Люди используют одни слова, но вкладывают разный смысл. Система делает эту разницу видимой.

## 2026-03 (середина) — Формализация: density matrices

**Поворот:** От наивных списков содержимого к операторам плотности из квантовой информатики.

**Почему density matrices, а не граф?**
- Polysemy = mixed state (von Neumann entropy)
- Asymmetric containment через Tr(ρ_A·ρ_B)/Tr(ρ_A)
- Eigendecomposition → грани смысла без предобучения
- Каждый компонент = rank-1 проектор с provenance (обратимость к исходному суждению)

**Реализовано:** `density_core.py` — Judgment, Component, Concept, SemanticSpace. Рекурсивное углубление ρ_deep.

**Осознание:** Density matrices — не "квантовая семантика" в физическом смысле. Это *инженерное решение*: data structure с полезными свойствами (спектральное разложение, асимметричное вхождение, энтропия как мера полисемии).

## v0.1 — Плоский экстрактор (baseline)

**Метод:** Итерация по токенам → VERB → nsubj/obj из прямых потомков.
**Результат:** 1026 суждений. "Человек" = хаб (44 как субъект). Отрицание плоское, модальность игнорируется, координация теряет второй элемент.
**Вывод:** Плоское извлечение непригодно — теряет структуру языка.

## v0.2 — Рекурсивный экстрактор

**Поворот:** Переход от плоского к рекурсивному обходу дерева зависимостей.

**Что появилось:**
- ExtractionContext: negation scope, modality, quantifiers, conditionals
- Neg-raising vs non-neg-raising глаголы

**Результат:** 1013 суждений (−1.3%). Количество примерно то же, но *качество* принципиально выше: отрицание корректное, модальность дифференцированная.

## v0.3 — Контекстный классификатор

**Поворот:** Классификация *до* извлечения. Не всё, что человек говорит, определяет его смыслы.

**Ключевое осознание:** "Этот человек вчера пришёл" — это эпизод, не определение. "Человек смертен" — это убеждение. Нужно разделять ДО записи.

**Два уровня классификации:**
1. Clause: GENERIC / HABITUAL / EPISODIC
2. NP: CONCEPTUAL / SELF / INTERLOCUTOR / GENERIC / REFERENTIAL / QUOTE

**Результат:** 877 суждений (−13.4%). "Человек" перестал быть хабом. Обобщённое "ты" теперь извлекается.

**Сигналы:** Аспект (НСВ/СВ) — ключевой для русского, ILP/SLP предикаты, детерминеры, темпоральные маркеры.

## 2026-03-27 — Критический пересмотр оснований

**Триггер:** Желание критически оценить фундамент — корректно ли выбрана стратегия картографирования персональных смыслов.

**Четыре фундаментальных вопроса и решения:**

### F1. Стенографический принцип — иллюзия чистоты
**Осознание:** Чистая стенография уже нарушена — context_classifier совершает инференции (обобщённо-личное "ты", GENERIC по аспекту). Это не дефект, а неизбежность.

**Решение:** Слоистая модель интерпретационной уверенности:
- L0 — прямое извлечение из синтаксиса (жёсткое ядро)
- L1 — лингвистические инференции (отменяемые)
- L2 — синергия с Антуражем (отменяемые, маркированные)

Каждое суждение маркируется `interpretation_layer`, `defeasible`, `inference_chain`.

**Опора:** Grice (1975) said/implicated; Default Logic (Reiter 1980); AGM belief revision (1985).

### F2. Проблема заземления
**Осознание:** Базовые вектора из spaCy = коллективные, не персональные. Для малого корпуса карта отражает коллективный язык, а не индивида.

**Решение:** "Персональный прогрев" — PCA по контекстам при >50 суждениях для понятия. Перспектива: гиперболическое вложение (Nickel & Kiela 2017).

**Опора:** Saussure (langue vs parole); Derrida (différance).

### F3. Система как модель, не зеркало
**Осознание:** "Могу объяснить → могу формализовать" — верно, но с оговорками. Dreyfus: embodied knowledge; Polanyi: tacit knowledge. FVSC записывает *нарратив* (как человек объясняет себя), а не *механизм*.

**Решение:** Позиционировать как полезную модель (Box 1976). Добавить `extraction_confidence`.

### F4. Забывание со следами
**Осознание:** Накопление без забывания не отражает динамику смыслов. Но ничего не должно исчезать бесследно.

**Решение:** Степенной decay `(1+Δt/τ)^{-d}` вместо экспоненциального. Архивация слабых компонентов (provenance сохранён). Консолидация через повторное подтверждение.

**Опора:** ACT-R (Anderson 1993); MINERVA 2 (Hintzman 1988); CLS (McClelland et al. 1995).

**Результат:** Whitepaper расширен до ~2250 строк. Новые разделы XVII (Эпистемологические основания) и XVIII (Временная динамика). 21 новый академический источник. Обновлены "Актуальные вопросы" (F1–F4 + T1–T9).

## 2026-03-27 — Имплементация: v0.5

**Реализованы три задачи в `density_core.py`:**

### F1 → код: слоистая модель в Judgment
Добавлены поля `interpretation_layer` (0/1/2), `defeasible`, `inference_chain`, `extraction_confidence`. Новый метод `Concept.rho_layer(max_layer)` строит ρ только из компонентов нужного слоя. Теперь можно "отключить" L1/L2 и увидеть чистое ядро.

### F3 → код: степенной decay + консолидация + архивация
- `_recompute_rho()` теперь применяет степенной decay: `w(t) = w₀ · activation_count · (1+Δt/τ)^{-0.5}`
- Компоненты ниже порога `ε·max(w)` → `archived=True` (в ρ не участвуют, provenance сохранён)
- `add_component()` теперь проверяет косинусное сходство: если >0.85, не добавляет дубликат, а увеличивает `activation_count` существующего компонента
- `reactivate()` восстанавливает из архива

### T6 → код: сравнение карт
`SemanticSpace.compare_maps(space_a, space_b)` — для каждого общего понятия считает `Tr(ρ_A^norm · ρ_B^norm)`. Возвращает divergent/aligned/global_similarity.

**Тесты:** test_poc.py проходит. Ручные тесты decay, консолидации, сравнения карт, фильтрации по слоям — все ок.

### F2 → код: персональный прогрев
`get_term_vector()` теперь трёхуровневый: если у понятия >50 компонентов — строит персональный seed через PCA (SVD по накопленным контекстам) вместо spaCy. Метод `_personal_basis()` возвращает первую главную компоненту. spaCy используется только как инициализация до набора персональных данных.

### T5 → код: relation-dependent transform для recursive deepening
`_relation_transform(ρ, relation)` — similarity transform `R_r·ρ·R_r†`, где R_r — ортогональная матрица, детерминированная от глагола связи. Сохраняет trace и eigenvalues, но поворачивает грани смысла в зависимости от отношения. `_dominant_relation(A, B)` находит самый частый глагол, связывающий A→B в суждениях. Recursive deepening теперь учитывает *через что* A содержит B.

### T10 → код: интерактивный канал обратной связи (FeedbackEngine)
Новый модуль `core/feedback.py`. Система перестала быть пассивной — появился третий канал входа (наряду с текстовым анализом и тезаурусом): интерактивная калибровка через диалог с Антуражем.

**FeedbackEngine** анализирует SemanticSpace и генерирует приоритизированные мягкие вопросы:
- Аномалии (prio=0.9): "Это что-то новое, или я неправильно понял контекст?"
- Противоречия (0.85): "Тут есть нюанс. Что ближе к правде?"
- Defeasible L1/L2 (0.6): "Я предположил X. Это похоже на правду?"
- Архив (0.5): "Раньше было важно X. Это по-прежнему так?"
- Контрасты по Kelly (0.3): "Чем X отличается от Y для тебя?"
- Milestones (0.2): "Я начинаю понимать, что для тебя значит X"

Ответы обновляют карту: подтверждение → consolidation, отклонение → архивация, промоция L1→L0.

В Judgment добавлены: `confirmation_status` ("unreviewed"/"confirmed"/"rejected"/"contextualized"), `context_tags` (контекстуальные убеждения).

**Принципы дизайна:** никогда не обвинять, всегда можно пропустить, непрямые exploratory вопросы, gamified milestones.

## 2026-04-07 — Сегментация предложений: v0.7.3

**Проблема:** Telegram-сообщения приходят без пунктуации. `live_test.py` мержит последовательные сообщения в блоки (45s окно), spaCy получает слитный текст без точек → `doc.sents` считает весь блок одним предложением → дерево зависимостей ломается → extraction теряет суждения из второго/третьего предложения. Тихий false negative, невидимый в evaluation (GOLD_STANDARD тестирует только одиночные чистые предложения).

**Решение:** WTSplit/SaT (Segment any Text, ACL 2024) — нейросетевой punctuation-agnostic сегментатор как L1 препроцессор.

**Архитектурный выбор:**
- Сегментация предложений лингвистически универсальна — это не про персональную семантику, а про общую грамматику. Поэтому нейросеть на этом уровне приемлема (в отличие от extraction, где важен персональный контекст).
- SaT классифицирует позиции как "граница/не граница" — не генерирует и не меняет текст.
- Детерминистичен при инференсе (нет рандомности).
- Защита от разрыва модальных конвертов: сегменты, начинающиеся с "что/чтобы/как/который", склеиваются обратно с предыдущим.
- Graceful degradation: если wtpsplit не установлен, fallback на split по newline (старое поведение).

**Принцип минимальной инвазии:** tree_extractor.py и text_normalizer.py не изменены. SaT работает upstream — подаёт spaCy уже отдельные предложения. `segment_and_flatten()` реплицирует timestamps для сохранения alignment contract.

**Временное решение:** Будет заменено при реализации T11 (морфологическое ядро), где pymorphy2 + падежная грамматика определят границы клауз детерминистически.

**Новый пайплайн:**
```
raw blocks (merged 45s) → segment_sentences(SaT) → per-sentence: normalize_text() → spaCy → _walk() → Judgment[]
```
