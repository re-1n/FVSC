## Фундаментальные вопросы

### F1. Слоистая модель интерпретационной уверенности

**Проблема:** Стенографический принцип ("записываем только эксплицитное") уже нарушен — context_classifier совершает инференции (обобщённо-личное "ты", классификация GENERIC по аспекту). Чистый L0 невозможен.

**Решение:** Три явных слоя интерпретации:
- **L0** — прямое извлечение из синтаксиса (текущий пайплайн)
- **L1** — инференции по лингвистическим и "общечеловеческим" правилам, всегда `defeasible=True`
- **L2** — синергия с Антуражем (LLM), всегда `defeasible=True`, `source="antourage"`

**Архитектура:** Добавить в Judgment: `interpretation_layer`, `inference_chain`, `defeasible`. Density matrix считать отдельно по слоям.

**Теоретическая опора:** Grice (1975) said/implicated; Default Logic (Reiter 1980); AGM belief revision (Alchourrón et al. 1985); Kahneman dual-process theory.

**Риск:** L1 ("общечеловеческие принципы") культурно нагружен. Nisbett & Wilson (1977): люди систематически ошибаются в интроспекции.

---

### F2. Проблема заземления (grounding) — **Решено (v0.5)**

**Проблема:** Базовые вектора из spaCy `ru_core_news_md` = коллективные, не персональные.

**Решение (реализовано):** `get_term_vector()` теперь трёхуровневый: персональный PCA (при >50 компонентах) → spaCy seed → детерминированный хеш. Метод `_personal_basis()` строит первую главную компоненту по SVD накопленных контекстов.

**Перспектива (не реализовано):** Гиперболическое вложение (Nickel & Kiela 2017) — v2+.

---

### F3. Забывание со следами (степенной decay)

**Проблема:** `recency = exp(-λ·Δt)` описан в whitepaper, но НЕ реализован в коде. Экспоненциальный decay слишком агрессивен — убьёт фундаментальные убеждения. Без decay карта бесконечно растёт.

**Решение:**
- Степенной decay: `w(t) = w₀ · (1 + Δt/τ)^{-d}`, d ≈ 0.5 — длинный хвост, ничего не обнуляется
- Архивация: компоненты с `w < ε·max(w)` → архив (provenance сохранён, но не в ρ)
- Консолидация (CLS-inspired): повторное подтверждение суждения усиливает base activation
- Per-user τ: начальное = median(inter-message-interval) × 100

**Теоретическая опора:** ACT-R (Anderson 1993) — степенной, не экспоненциальный закон памяти; MINERVA 2 (Hintzman 1988) — отдельные следы; Complementary Learning Systems (McClelland et al. 1995) — гиппокамп → неокортекс.

**Файл:** `core/density_core.py` — Component, `_recompute_rho()`.

---

### F4. Система как модель, не зеркало

**Проблема:** Тезис "могу объяснить → могу формализовать" ограничен. Dreyfus (1972): embodied knowledge не формализуемо. Polanyi (1966): "знаем больше, чем можем сказать". Текст = тонкий срез смысла.

**Решение:** FVSC записывает как человек *объясняет себя*, не как он устроен. Это карта нарратива, не механизма — и для целей проекта это корректно. Box (1976): "All models are wrong, but some are useful."

**Практика:** Добавить `extraction_confidence` к каждому суждению. Не позиционировать как "зеркало сознания".

**Теоретическая опора:** Marr (1982) три уровня анализа; Fodor (1975) Computational Theory of Mind; Nisbett & Wilson (1977).

## Вопросы качества кода (ревью 2026-03-28)

Сессия v0.5.1: исправлены 7 критических/средних багов. Ниже — оставшиеся вопросы, требующие решения.

### C1. Type safety в density_core.py — `rho` возвращает `None`, аннотирован как `np.ndarray`

**Где:** `density_core.py:93, 102, 113, 122` — properties `rho`, `rho_norm`, `rho_deep`, `rho_deep_norm`

**Проблема:** Все четыре property аннотированы `-> np.ndarray`, но возвращают `None` когда нет активных компонентов. Код-ревьюер или mypy не предупредит вызывающий код о необходимости проверки `None`.

**Решение:** Заменить на `-> Optional[np.ndarray]`. Уже есть проверки `if r is None` внутри `rho_norm` и `rho_deep_norm`, но аннотации врут.

**Приоритет:** Средний. Не крашит сейчас (все вызывающие уже проверяют), но создаёт ловушку для нового кода.

---

### C2. `_is_verb` — monkey-patched атрибут вместо поля Concept

**Где:** `density_core.py:494` — `verb_concept._is_verb = True`; читается через `getattr(c, '_is_verb', False)` в нескольких местах

**Проблема:** Атрибут не объявлен в dataclass. Не переживёт сериализацию/десериализацию, невидим для type checker, не появляется в `__repr__`.

**Решение:** Добавить `is_verb: bool = False` в `Concept`. Установить в `materialize_judgment` вместо monkey-patch.

**Стоит ли:** Да, простое исправление. Но нужно убедиться что все `getattr(c, '_is_verb', False)` заменены на `c.is_verb`.

---

### C3. `Judgment.timestamp = 0.0` — ловушка для вызывающего кода

**Где:** `density_core.py:30`

**Проблема:** Default timestamp = 0.0 (Unix epoch, 1970). Если вызывающий код забудет установить timestamp, компонент получит ~56 лет decay и фактически нулевой вес. Сейчас `tree_extractor` не устанавливает timestamp вообще — все Judgment'ы имеют `timestamp=0.0`.

**Решение:** Два варианта:
1. `timestamp: float = field(default_factory=time.time)` — авто-заполнение текущим временем
2. Установить timestamp в `tree_extractor.py` при создании Judgment (более явно)

**Стоит ли:** Да, это реальная проблема. Вариант 1 проще, но вариант 2 корректнее (timestamp сообщения, не момент обработки). Для Telegram-данных timestamp должен быть из поля `date` сообщения.

**Приоритет:** Высокий. Сейчас decay работает некорректно для всех суждений из tree_extractor.

### C4. `_queryable_concepts` считает архивированные компоненты

**Где:** `density_core.py:590`

**Проблема:** `len(c.components) >= self.min_components` включает архивированные. Concept с 10 архивированными и 0 активными пройдёт проверку, но `rho_deep` вернёт `None` (ловится следующей проверкой, но лишняя работа).

**Решение:** `sum(1 for c in c.components if not c.archived) >= self.min_components`

**Стоит ли:** Да, тривиальный фикс. Но пока decay не работает (C3), архивации не происходит, и баг латентный.

**Проблема:** Функция обрабатывает consequent-клаузы условных конструкций, но пропускает: passive voice inversion, discourse pointer filtering, INTERLOCUTOR remapping, GENERIC modality reduction для субъектов. Результат: условные конструкции обрабатываются менее качественно.

## Архитектурные вопросы (сессия 2026-03-29/30)

### T11. Морфологическое ядро — замена spaCy dep-parse

**Проблема:** spaCy dep-parser — нейросеть в сердце L0. 7-12% ошибок на чатах попадают в L0 как неотменяемые (`defeasible=False`). Стенографический принцип нарушен фактически при формальной чистоте. Система привязана к UD-схеме, спроектированной для аннотаторов, не для понимания смысла.

**Ключевое наблюдение:** Русский — флективный язык. Падежная система **уже кодирует синтаксические роли** в окончаниях слов: именительный = субъект, винительный/родительный = объект. spaCy решает задачу, которая для русского языка в значительной мере уже решена грамматикой.

**Решение — трёхуровневый детерминированный экстрактор:**

1. **pymorphy2** (rule-based) — морфологический разбор: лемма, падеж, аспект, наклонение, род, число. Детерминированный, не нейросеть.

2. **Падежная грамматика + словарь валентностей** — формальные правила:
   ```
   требовать(NOM=Agent, GEN=Patient)
   любить(NOM=Agent, ACC=Patient)
   помогать(NOM=Agent, DAT=Beneficiary)
   нравиться(DAT=Experiencer, NOM=Theme)
   ```
   Источник валентностей: Апресян (1995), НКРЯ.

3. **Самодизамбигуация через накопленный контекст** — при морфологической омонимии система выбирает вариант с наименьшим `anomaly_score` по существующей ρ. Трассируемо: `inference_chain: ["case_ambiguity:NOM/ACC", "resolved_by:anomaly:0.15<0.82"]`. Улучшается с данными без обучения.

**Теоретическая опора:**
- Fillmore, C. (1968). "The Case for Case." — Падежная грамматика: глубинная структура = падежные роли.
- Mel'čuk, I. (1974). "Opyt teorii lingvisticheskikh modelej Smysl↔Tekst." — MTT: формальные уровни между смыслом и текстом.
- Apresjan, Ju.D. (1995). "Integral Description of Language and Systemic Lexicography." — Валентности русских глаголов.
- Zaliznyak, A.A. (1977). "Грамматический словарь русского языка." — Полные морфологические парадигмы.
- Paducheva, E.V. (2004). "Dynamic Models in the Semantics of Lexicon." — Аргументная структура русских глаголов.

**Работает даже на выдуманных словах:** "Глокая куздра штеко будланула бокра" (Щерба) — pymorphy2 определяет роли по окончаниям: "-ая" = прилаг. жен.р., "-а" = им.п. жен.р., "-ула" = глагол прош.вр. жен.р., "-а" в "бокра" = вин.п.

**spaCy остаётся** как опциональный L1 (для неоднозначных случаев, fallback). LLM — L3.

**Приоритет:** Высокий. Это фундаментальная перестройка L0.

### T12. Универсальная архитектура

**Проблема:** Текущая реализация привязана к русскому языку. Но density matrix core (контейнеры, containment, recursive deepening, anomaly detection) — полностью универсальна. Языкоспецифичен только входной слой (extraction).

**Решение — pluggable language adapter:**

```
Текст (любой язык)
    ↓
[Языковой адаптер]  ← единственная языкоспецифичная часть
  - русский: pymorphy2 + падежи + валентности
  - английский: word order + function words
  - японский: particle detection (は, を, に)
    ↓
Judgment (S → V → O)  ← универсальный формат
    ↓
[Density matrix core] ← полностью универсальное
    ↓
Семантическая карта
```
